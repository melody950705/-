# PRD: 台中等公車2.0

## 1. 專案概述
### 1.1 背景與動機
台中市民與學生以公車作為交通工具之一，而現有的公車系統常出現時間不準確、公車忽然在系統上消失、超過發車時間卻沒有發車等問題。此外，使用者在查詢公車到站時間、路線轉乘與附近站牌時，常需要花費較多時間搜尋，嚴重影響通勤效率。因此，我們希望設計一套更簡潔、有效率、更新即時且符合學生與通勤族需求的公車查詢系統「台中等公車2.0」，提升台中市民日常搭乘公車的便利性與效率。

### 1.2 目標用戶
- 台中市民
- 學生
- 通勤族
- 對公車有需求的人（如遊客）
- 公車司機

### 1.3 核心價值主張
- 提升資訊即時性與準確度
- 建立公車司機即時回報機制，降低公車行駛錯誤率
- 改善現有系統操作複雜問題，提升使用體驗
- 整合下車提醒功能，避免錯過站牌
- 最終達成提升大眾運輸利用率的目標

## 2. 功能需求

### 2.1 主要功能與使用者故事

1. **即時公車動態查詢**
   - 作為 **公車乘客**，我希望 **能即時查詢公車的預估到站時間與當前位置**，以便 **我能準確規劃出門時間，減少在站牌等待的時間**。

2. **公車司機即時回報系統**
   - 作為 **公車司機**，我希望 **能有專屬權限快速回報路況、車輛故障或發車延誤狀況**，以便 **系統能即時更新動態資訊給乘客，降低資訊落差**。

3. **智慧下車提醒功能**
   - 作為 **通勤族/學生**，我希望 **能在快到目的站點時收到手機提醒**，以便 **我可以在車上安心休息或處理事情，不用擔心坐過站**。

4. **簡潔的路線與轉乘規劃**
   - 作為 **對公車有需求的人**，我希望 **能輸入起迄點並獲得最快或最少轉乘的公車路線建議**，以便 **我在不熟悉的地點也能輕鬆找到正確的乘車方案**。

5. **附近站牌與公車搜尋**
   - 作為 **公車乘客**，我希望 **能一鍵搜尋我目前位置附近的公車站牌與即將到站的路線**，以便 **我能快速決定要走到哪個站牌搭車**。

## 3. 非功能需求
### 3.1 技術限制
- 後端框架：Flask (Python)
- 前端技術：Jinja2 Template + HTML/CSS/JS
- 資料庫：SQLite

### 3.2 效能與安全考量
- **效能**：查詢公車即時動態的 API 必須快速回應，避免長時間等待（建議回應時間 < 1 秒）。
- **安全**：司機回報系統需有權限驗證機制（登入），避免一般使用者隨意竄改行車狀態。
- **資料即時性**：整合政府開放資料 (如 TDX) API，確保資料同步率。

## 4. MVP 範圍 (Minimum Viable Product)

### 4.1 Must Have (必須擁有 - 核心功能)
- 即時公車動態查詢（結合開放資料 API）
- 附近站牌與公車搜尋
- 公車司機登入與基本回報功能（例如：延誤、滿載、故障）

### 4.2 Should Have (應該擁有 - 重要功能)
- 簡潔的路線與轉乘規劃
- 智慧下車提醒功能（基於時間或 GPS 預估）

### 4.3 Nice to Have (可以擁有 - 加分功能)
- 常用路線/站牌收藏功能
- 司機端 GPS 輔助定位回報
- 黑夜模式 (Dark Mode) UI 設計
